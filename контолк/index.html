Программа и инфраструктура для её работы
Название: Kontalk;
Происхождение: Свободное сообщество;
Статус: Активна, доступна для регистрации и бесплатна;
Версии: Android, iPhone, BlackBerry, ПК (Windows, Mac OS X, Linux);
Тип: XMPP (Jabber);
Особенности: RSA шифрование, телефонный номер является идентификатором.

Текущая версия (под Android): 4.0;


Коротко


Контолк (или Конталк) (англ. Kontalk) — программа для смартфонов с ОС Android, а также iPhone. Использует в качестве основы протокол XMPP, то есть Jabber. В качестве идентификаторов пользователей используются телефонные номера, которые берутся из телефонной книги Android. Сами номера не передаются на сервер, а передаются только их хэш-суммы. После этого зарегистрированные в Kontalk контакты автоматически появляются в Kontalk и с ними можно начать разговор.

Основным разработичиком программы Kontalk для Android является итальянский программист Даниэлле Риччи.

Шифрование

Kontalk следует принципу, что у граждан должно быть конституционное право на приватную переписку, поэтому шифрует отправленные сообщения. Сообщения отправляются как plain текст, на который наложено 256 RSA шифрование. Чтобы установить шифрованную переписку с контактом, нужно подтвердить его ключ. Отправить ему сообщение "Тест" и у него появится меню, сообщающее, что шифрованное соединение не установлено. Для того, чтобы установить шифрованное соединение, нужно обоюдно верифицировать ключ.

Откройте верификацию ключа и его публичная часть отобразится на экране. Позвоните собеседнику или напишите ему электронное письмо, спросив его, действительно ли ключ такой, какой Вы видите на экране. Если всё соотствествует действительности, то подтвердите верность ключа, после чего выждите 2 минуты, нажмите кнопку написания нового сообщения и обновите список контактов (потяните список вниз, после чего появится сообщение, гласящее о том, что список был обновлён).


Использование на нескольких устройствах

Kontalk автоматически загружает вашу переписку. Для того, чтобы это сделать, вам нужно экспортировать свой ключ. Для этого зайдите в меню Настройки -> Обслуживание -> Экспорт ключевой пары. Придумайте пароль. Запишите его на тайный листочек, чтобы случайно не забыть. Постарайтесь также не потерять этот листочек. Экспортируйте ключевую пару в папку Downloads на устройстве. Затем откройте файловый менеджер и передайте созданный программой zip архив на своё второе устройство (например, со смарфтона на планшет). На втором устройте вместо регистрации откройте меню и выберите пункт "импорт ключа". Откройте принятый файл с ключом и введите придуманный вами пароль от него. Произойдёт вход в Kontalk, после чего загрузятся ваши сообщения и настойки. Можно начинать использование в штатном режиме.


Возможности программы

  • Общение с контактами Kontalk с помощью сообщений;
  • Возможность поделиться фотографией с Kontalk контактом через меню "Поделиться", находящимся в программе-галерее;
  • Отправка голосовых сообщений (формат 3gp);
  • Поддержка старых версий Andorid (минимум 2.3);
  • Оффлайн доставка сообщений;
  • Смайлики и пиктограммы (множество);
  • Отправка фотографий и доставка их в оффлайн;
  • Статусы;
  • Группы пользователей (сделано так, что непригодно для использования: нужно запомнить команды, а потом вводить их в чат);
  • RSA шифрование сообщений (выполнено через XEP имплементацию 0374);
  • Связь с обычными Jabber контактами (при условии, что добавление контакта произведут с их стороны);
  • Недописанное сообщение сохраняется в черновик.
  • Kontalk работает при нестабильном соединении. Чуть только Интернет-соединение появится на короткий промежуток времени — контолк отправит Ваше сообщение на сервер. А оттуда оно уже дойдёт адресату сразу же, как это станет возможным. Если он в сети, то это пройзойдёт всего через несколько секунд.

Отличия от Telegram

  • Вы видите публичную часть своего ключа, так же QR код (а в Telegram не видите). При желании можно посмотреть и личную часть ключа через базу данных.;

  • Безопаснее: управляется сообществом;
  • XMPP стандарт (а в Telegram своя технология);
  • RSA шифрование plain текста (а в Telegram своя технология ширования — MTProto и она более слабая);
  • Сообщения доставляются на пару-тройку секунд дольше, чем в Telegram (в Telegram немного быстрее);
  • Гораздо экономичнее для батареи, чем Telegram (а Telegram сажает батарею куда сильнее);

  • Отсутствует двухфакторная ауентификация. Но исключена возможность кражи сообщений при получении доступа к мобильнику. Поменяется ключ, а сообщения в хранилище станут недоступными. (В Telegram присутствует двухфакторный механизм авторизации, он включается в настройках.);
  • Очень много смайликов и пиктограмм (в Telegram поменьше, но там можно создавать стикеры);
  • Отравленные фотографии можно не пережимать, а можно настроить сжатие, есть 3 варианта сжатия: большой, средний и малый (а в Telegram СЖИМАЮТСЯ);
  • Передаются только хэши телефонных номеров (а в Telegram можно и без номера писать собеседнику);
  • Небольшое по размеру (10,8 Мб) приложение и мало места в оперативной памяти занимает (у меня это 35 Мб). А вот Telegram почти в 2 раза тяжелее и отнимает куда больше оперативки;
  • На русском языке, лично я доделал перевод (а Telegram нужно переводить дичайшим способом — через отправку себе языкового файла)
  • Возможность быть Онлайн постоянно (а в Telegram Онлайн статус спадает через минуту);
  • Нельзя отправлять любые файлы, нельзя отправить музыку (а в Telegram можно) — это по той причине, что у программистов кривые руки;
  • Прозрачен для пользователя (а Telegram что-то где-то шифрует, но Вы не в курсе);
  • Аватары в программе используются из списка контактов. Если аватара нет, то показывается первая буква имени в цветном кружке. (А в Telegram у каждого они загружаются на сервер)
  • Можно посмотреть свойства сообщения — его размер и параметры (а в Telegram нельзя);
  • Можно связываться с Jabber контактами (а в Telegram нельзя). Теперь также можно избирательно отключать шифрование;
  • Нет web-версии;
  • Работает на Jolla (если включить доступ к контактам для Android-приложений). Telegram также работает на Jolla, имеется нативный клиент;
  • Отсутствует облачная синхронизация, ибо нет облака (а в Telegram есть);
  • Никаких проблем с прокруткой истории назад (а Telegram долго грузит её, а вот web-версию вообще коматозит).

Вопрос — ответ

Вопрос: Некоторые вечерние сообщения отображаются как отправленные, но непрочитанные, хотя были написаны ещё до выхода собеседника в оффлайн. Почему так происходит?
Ответ: Потому что у контакта ей таймаут на статус при потере соединения, а на сообщение шлëтся колбэк, и он не доходит, потому что у тебя рвëтся соединение. Или это баг. Скорее первое.

Вопрос: Почему неопределённый статус у контакта?
Ответ: Это значит, что контакт испытывает проблемы с соединением. Соединение недостаточно стабильное, чтобы обозначить его как онлайн, но и он периодически шлёт пакеты, подтверждая свою некоторую активность. Связь с таким собеседником будет нарушена до тех пор, пока он не перестанет испытывать проблемы с соединением.

Вопрос: Почему статус онлайн написан курсивом?
Ответ: Потому что вы находитесь в зоне нестабильного соединения. Или ваш контакт находится в зоне нестабильного соединения.

Вопрос: Как проверить наличие соединения?
Ответ: Напишите сообщение самому себе. Должен прийти ответ с таким же текстом, какой был в отправленном. Если пришёл ответ от себя, значит всё работает. В новой версии также можно увидеть активность себя, просто открыв переписку с собой, но это недостаточно точная проверка.

Вопрос: Почему показывается, что контакт был в сети полчаса назад, хотя сообщение от него я получил минуту назад?
Ответ: Потому что Даниэлле наговнокодил. Радуйтесь, что сообщения вообще доходят.


Предостережения

  • Не забывайте, что это приложение для личной переписки, думайте дважды перед тем как давать телефон/планшет кому-то поиграть. Или используйте профилирование, которое добавили в Android 6.

Использование на нескольких устройствах без kontalk-keys.zip

Способ работает только на Android 4! Для совершения этого трюка вам понадобятся: root доступ на обоих смартфонах/планшетах (Super SU), Titanium Backup, Root Explorer. И сама программа Kontalk. Первым делом найдите файл accounts.db. На Android устройтве этот файл находится в папке /system/users/0. Его нужно передать на второе устройство.

Просто передайте этот файл по Bluetooth. Теперь поместите этот файл по тому же пути на втором устройстве с заменой.

Второй способ: перенести оттуда строки в базу второго устройства. Он нуждается в пояснениях. Установите Kontalk на втором устройстве и заморозьте его в Titanium Backup. Теперь вам понадобится программа SQLite Editor. Откройте в ней accounts.db на первом устройстве.

В списке аккаунтов найдёте Kontalk. Пароль к нему хранится в MD5. В extra вы найдёте ещё строки с ключами. При желании Вы можете всё это скопировать в БД на втором устройтве. А теперь задайтесь вопросом — нахрен это надо?

После выполненных манипуляций сделайте бэкап программы Kontalk в Titanium Backup и передайте этот бэкап на второе устройство. На нём уже должен быть подготовленный свежий accounts.db. Теперь активируйте восстановите данные из бэкапа Kontalk. Затем перезагрузите устройство. После загрузки устройства разморозьте Kontalk в Titanium Backup. Всё. Можно пользоваться. Может нифига не сработать.

Компиляция программы для ПК

Разработчик Abika вносит улучшения в код, согласно замечаниям пользователей. Но новую версию ему собирать почему-то не хочется. Поэтому он предлагает нам собрать её самим. Однако, программа собирается очень неудобным способом: фреймворк собирает необходимые компоненты, а затем компилирует программу, после чего запускает её. После закрытия программы можно найти лишь код, а программу нужно компилировать заново для следующего запуска. Для того, чтобы скомпилировать программу, нам необходимо проделать несложные действия. Информация взята из официального FAQ, но снабжена подробностями. Все действия, кроме установки самой системы git (только первого шага) производятся от не суперпользователя. Приступим:

  • Установите git из репозиториев своего дистрибутива. Это единственный шаг, для которого понадобятся права суперпользователя;
  • В терминале выполните
    git submodule update --init
    ;
    • git submodule update --init
  • В терминале выполните
    git clone https://github.com/kontalk/desktopclient-java/
    из папки, куда Вы обычно складываете GIT проекты;
  • Дождитесь окончания загрузки. Откройте терминал из папки desktopclient-java;
  • Выполните команду
    ./gradlew run
    Примерно на 48 секунде запустится свежий Kontalk.
  • После закрытия Kontalk будет обозначено 100% и выполнение кода прекратиться. Компиляцию кода нужно производить перед каждым запуском программы.
